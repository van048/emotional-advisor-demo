# 情感陪伴 产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品名称
**情感陪伴** - 基于情绪感知的智能咖啡推荐系统

### 1.2 产品定位
将咖啡机从功能性设备升级为"情绪疗愈伙伴"，通过AI理解用户当下情绪状态，推荐最契合的咖啡制作方案，在快节奏生活中提供有温度的陪伴体验。

### 1.3 核心价值
- **用户价值**：简化决策成本，让咖啡制作成为情绪调节的仪式感
- **业务价值**：提升咖啡机使用频次，构建品牌情感护城河
- **数据价值**：积累情绪-行为关联数据，优化推荐算法

---

## 2. 功能架构

### 2.1 范围界定（MVP）

| 维度 | 范围 |
|:---|:---|
| **平台** | H5页面，嵌入现有App |
| **设备** | 仅支持咖啡机（单点打透） |
| **情绪** | 4种一级标签：疲惫、开心、焦虑、想静一静 |
| **推荐** | 单食谱推荐，AI动态生成 |
| **交互** | 展示完整步骤，一键启动全自动制作，仅跟踪整体状态 |

### 2.2 功能模块

```
情感陪伴
├── 情绪层：4种情绪主动选择
├── 推荐层：AI生成单食谱 + 3句话推荐理由 + 步骤预览
├── 执行层：一键启动，咖啡机自动完成，仅反馈整体进度
└── 状态层：整体制作状态（未开始/制作中/已完成/异常）
```

## 3. 详细功能设计

### 3.1 情绪选择

**页面结构**：
- 标题："此刻，你的心情是？"
- 4个情绪卡片，网格布局（2x2）
  - 疲惫 😔：深灰色调，"身体需要被温柔对待"
  - 开心 😊：暖黄色调，"让快乐再延续一会儿"
  - 焦虑 😰：紫色调，"深呼吸，慢慢来"
  - 想静一静 😌：青绿色调，"给自己一段独处时光"
- 底部提示："选择后，AI会为你推荐最适合的咖啡方案"

**交互**：点击卡片进入推荐页，无二次确认。

---

### 3.2 AI推荐引擎

#### 3.2.1 输入维度
| 维度 | 数据来源 | 示例 |
|:---|:---|:---|
| 用户情绪 | 用户主动选择 | 疲惫 |
| 时段 | 系统时间 | 22:30（深夜） |
| 季节/天气 | 系统时间+定位天气 | 冬季/雨天 |
| 设备能力 | 咖啡机物模型 | 支持意式/美式/奶咖，支持设置若干参数 |

#### 3.2.2 输出内容
- **食谱名称**：如"低因热拿铁·晚安版"
- **推荐理由**（3句话）：
  > 深夜时分，低咖啡因避免影响睡眠质量。温热的牛奶融合浓缩咖啡，舒缓紧绷的神经。制作过程简单，5分钟后即可享用，让疲惫随着香气散去。
- **步骤列表**：见3.3节

#### 3.2.3 推荐逻辑（技术实现）
- 不使用硬编码映射表
- 大模型驱动（Prompt包含全部输入维度+食谱库）
- 返回结果需包含：食谱ID、步骤分解、推荐理由

---

### 3.3 步骤展示与一键执行

#### 3.3.1 步骤列表结构
展示完整的制作流程，仅作预览，不跟踪单步进度，示例：

| 步骤 | 内容 | 咖啡机参数 |
|:---|:---|:---|
| 1 | 研磨咖啡豆 | 15g，细研磨度 |
| 2 | 萃取浓缩咖啡 | 水温92℃，压力9bar，30ml，25秒 |
| 3 | 打发牛奶 | 200ml全脂牛奶，60℃，中厚度奶泡 |
| 4 | 融合出品 | 自动注入奶泡，形成拉花效果 |
| 5 | 保温 | 55℃，持续3分钟 |

**展示形式**：
- 时间轴样式，垂直排列
- 全部步骤默认灰色/默认样式
- 不实时高亮当前步骤

#### 3.3.2 页面结构

**推荐展示区**：
- 食谱名称（大标题）
- 3句话推荐理由
- 制作信息：总时长、杯量、温度、难度

**步骤预览区**：
- 标题："制作流程"
- 展开/折叠切换（默认展开）
- 时间轴形式展示全部步骤（静态，无实时状态）

**执行区**（底部固定）：
- 【开始制作】按钮（蓝色主按钮）
- 按钮上方提示："咖啡机将按上述步骤自动完成制作，请确认物料已准备就绪"

#### 3.3.3 交互流程

**点击【开始制作】**：
1. 弹窗二次确认："即将开始制作『低因热拿铁·晚安版』，预计5分钟。请确认咖啡豆、牛奶已就位。"
2. 用户确认后，按钮变为【制作中】（禁用状态，带旋转loading）
3. 下发完整参数至咖啡机，咖啡机自动执行全部步骤
4. 页面仅展示整体进度：总进度条 + "制作中，请稍候..."

**制作完成**：
- 咖啡机上报完成状态
- 页面显示"制作完成，请享用☕"
- 提供【再来一杯】和【返回首页】

**主动停止**：
- 制作中显示【停止】按钮（红色，位于【制作中】按钮旁）
- 点击→弹窗确认"确定停止制作？"
- 确认后发送停止指令，咖啡机暂停，状态变为"已暂停"
- 提供【继续制作】（从断点恢复）或【重新开始】

---

### 3.4 状态管理

仅跟踪整体制作状态，不跟踪具体步骤：

| 状态 | 页面表现 | 说明 |
|:---|:---|:---|
| 未开始 | 【开始制作】按钮可点击，步骤列表静态 | 默认状态 |
| 制作中 | 按钮禁用+loading，显示总进度条，步骤列表保持静态 | 咖啡机自动执行中 |
| 已完成 | 成功提示，按钮消失 | 制作结束 |
| 已暂停 | 显示【继续制作】按钮 | 用户主动停止或异常中断 |
| 异常 | 错误提示+【重试】按钮 | 设备报错（缺豆/缺水等） |

---

### 3.5 异常处理

| 异常场景 | 检测方式 | 处理方式 |
|:---|:---|:---|
| 设备离线 | API调用超时/心跳断开 | 【开始制作】置灰，提示"咖啡机已离线，请检查网络连接" |
| 缺豆 | 咖啡机启动前自检上报 | 弹窗提示"咖啡豆不足，请添加后重试"，停留在未开始状态 |
| 缺水 | 咖啡机启动前自检上报 | 弹窗提示"水箱缺水，请添加饮用水"，停留在未开始状态 |
| 制作中断 | 设备状态异常/用户物理按键 | 状态变为"已暂停"，提供【继续制作】 |
| 网络中断 | 请求失败 | Toast提示"网络异常"，制作继续执行，状态稍后同步 |

## 4. 数据埋点

### 4.1 功能渗透指标
| 事件 | 说明 |
|:---|:---|
| `emotion_select_{mood}` | 选择情绪（mood= tired/happy/anxious/calm） |

### 4.2 推荐效果指标
| 事件 | 说明 |
|:---|:---|
| `emotion_recipe_show` | 推荐页曝光（含AI生成耗时） |
| `emotion_step_expand` | 展开步骤列表（若默认折叠则记录） |
| `emotion_start_click` | 点击【开始制作】 |
| `emotion_start_confirm` | 二次确认后正式下发指令 |
| `emotion_make_complete` | 制作正常完成 |
| `emotion_make_stop` | 用户主动停止 |
| `emotion_make_error` | 异常中断 |

### 4.3 留存与商业指标
| 事件 | 说明 |
|:---|:---|
| `emotion_user_return_1d` | 使用后次日回访 |
| `emotion_user_return_7d` | 使用后7日回访 |
| `coffee_machine_use_cnt` | 咖啡机使用次数（对比实验组/对照组） |
| `emotion_session_duration` | 功能使用时长 |

## 5. 非功能需求

### 5.1 性能要求
- 页面加载：<2s（H5）
- AI推荐生成：<3s（含加载动画）
- 指令下发响应：<1s

### 5.2 兼容性
- 支持iOS 12+ / Android 8+ WebView
- 适配主流屏幕尺寸（320px-414px宽度）

### 5.3 安全
- 咖啡机控制指令需用户登录态校验
- 设备通信使用HTTPS/WSS加密

## 6. 附录

### 6.1 情绪-场景示例（供AI Prompt参考）

| 情绪 | 时段 | 天气 | 推荐方向 | 示例食谱 |
|:---|:---|:---|:---|:---|
| 疲惫 | 深夜 | - | 低因、温热、助眠 | 低因热拿铁·晚安版 |
| 疲惫 | 午后 | 雨天 | 温热、香甜、治愈 | 焦糖玛奇朵·雨天特调 |
| 开心 | 早晨 | 晴天 | 清爽、提神、仪式感 | 冰美式·晨间唤醒 |
| 焦虑 | 下午 | - | 低因、顺滑、安抚 | 燕麦拿铁·平缓心绪 |
| 想静一静 | 夜晚 | - | 手冲感、慢节奏、专注 | 单品手冲·独处时光 |

### 6.2 设备物模型要求
咖啡机需支持以下能力：
- 接收并解析完整制作参数（多步骤序列）
- 按内置程序自动执行全部步骤
- 整体状态上报：空闲/制作中/已完成/异常
- 异常自检：豆仓空、水箱空、堵塞等
- 远程启停控制，支持断点续作
