# 情感陪伴 产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品名称
**情感陪伴** - 基于情绪感知的智能咖啡推荐系统

### 1.2 产品定位
将咖啡机从功能性设备升级为"情绪疗愈伙伴"，通过AI理解用户当下情绪状态，推荐最契合的咖啡制作方案，在快节奏生活中提供有温度的陪伴体验。

### 1.3 核心价值
- **用户价值**：简化决策成本，让咖啡制作成为情绪调节的仪式感
- **业务价值**：提升咖啡机使用频次，构建品牌情感护城河
- **数据价值**：积累情绪-行为关联数据，优化推荐算法

---

## 2. 功能架构

### 2.1 范围界定（MVP）

| 维度 | 范围 |
|:---|:---|
| **平台** | H5页面，嵌入现有App |
| **设备** | 仅支持咖啡机（单点打透） |
| **情绪** | 4种一级标签：疲惫、开心、焦虑、想静一静 |
| **推荐** | 单食谱推荐，AI动态生成 |
| **交互** | 步骤化执行 |

### 2.2 功能模块

```
情感陪伴
├── 情绪层：4种情绪主动选择
├── 推荐层：AI生成单食谱 + 3句话推荐理由
├── 执行层：步骤列表，咖啡机步骤可独立控制
└── 状态层：步骤状态跟踪（未开始/执行中/已完成）
```

## 3. 详细功能设计

### 3.1 情绪选择

**页面结构**：
- 标题："此刻，你的心情是？"
- 4个情绪卡片，网格布局（2x2）
  - 疲惫 😔：深灰色调，"身体需要被温柔对待"
  - 开心 😊：暖黄色调，"让快乐再延续一会儿"
  - 焦虑 😰：紫色调，"深呼吸，慢慢来"
  - 想静一静 😌：青绿色调，"给自己一段独处时光"
- 底部提示："选择后，AI会为你推荐最适合的咖啡方案"

**交互**：点击卡片进入推荐页，无二次确认。

---

### 3.2 AI推荐引擎

#### 3.2.1 输入维度
| 维度 | 数据来源 | 示例 |
|:---|:---|:---|
| 用户情绪 | 用户主动选择 | 疲惫 |
| 时段 | 系统时间 | 22:30（深夜） |
| 季节/天气 | 系统时间+定位天气 | 冬季/雨天 |
| 设备能力 | 咖啡机物模型 | 支持意式/美式/奶咖 |

#### 3.2.2 输出内容
- **食谱名称**：如"低因热拿铁·晚安版"
- **推荐理由**（3句话）：
  > 深夜时分，低咖啡因避免影响睡眠质量。温热的牛奶融合浓缩咖啡，舒缓紧绷的神经。制作过程简单，5分钟后即可享用，让疲惫随着香气散去。
- **步骤列表**：见3.3节

#### 3.2.3 推荐逻辑（技术实现）
- 不使用硬编码映射表
- 调用大模型API（Prompt包含全部输入维度+食谱库RAG检索）
- 返回结果需包含：食谱ID、适配参数、步骤分解、推荐理由

---

### 3.3 步骤执行

#### 3.3.1 步骤列表结构
每个食谱拆解为3-5个步骤，示例：

| 步骤 | 类型 | 内容 | 操作 |
|:---|:---|:---|:---|
| 1 | 准备 | 准备200ml全脂牛奶，倒入奶缸 | 无按钮，纯文字 |
| 2 | **咖啡机** | 研磨15g咖啡豆，**细研磨度**，放入手柄压粉 | 【启动研磨】 |
| 3 | **咖啡机** | 萃取30ml浓缩咖啡，**水温92℃，压力9bar，时间25秒** | 【启动萃取】 |
| 4 | 手工 | 将牛奶加热至60℃，打发出细腻奶泡 | 无按钮，纯文字 |
| 5 | **咖啡机** | **保温模式55℃**，保持咖啡温度 | 【启动保温】 |

#### 3.3.2 咖啡机步骤交互

**初始状态**：
- 步骤卡片显示参数（加粗高亮）
- 右下角【启动】按钮，蓝色主按钮样式

**点击【启动】**：
1. 弹窗二次确认："即将启动研磨，请确认咖啡豆已放入豆仓"
2. 确认后，按钮变为【停止】（红色），步骤状态变为"执行中"
3. 调用咖啡机API，下发对应参数
4. 等待设备状态回传（轮询或WebSocket）

**执行完成**：
- 设备上报完成 → 步骤状态变为"已完成"（绿色打勾）
- 【停止】按钮消失

**主动停止**：
- 点击【停止】→ 弹窗确认"确定停止当前步骤？"
- 确认后，发送停止指令，步骤状态变为"已中断"（灰色），可重新点击【启动】

#### 3.3.3 用户自选顺序
- 步骤列表全部展开，无强制顺序锁定
- 用户可点击任意咖啡机步骤的【启动】
- 但建议按序号执行（文案提示"建议按顺序制作，口感更佳"）

---

### 3.4 状态管理

#### 3.4.1 步骤卡片状态
| 状态 | 视觉样式 | 说明 |
|:---|:---|:---|
| 未开始 | 白色背景，灰色边框 | 默认状态 |
| 执行中 | 蓝色边框，呼吸灯效果 | 咖啡机正在工作 |
| 已完成 | 绿色边框，打勾图标 | 该步骤已结束 |
| 已中断 | 灰色背景，感叹号图标 | 用户主动停止或异常中断 |

#### 3.4.2 页面级状态
- 顶部显示当前制作中的食谱名称
- 全部步骤完成后，显示"制作完成，享受你的咖啡时光☕"
- 提供【再来一杯】（返回情绪选择）和【返回首页】

---

### 3.5 异常处理

| 异常场景 | 检测方式 | 处理方式 |
|:---|:---|:---|
| 设备离线 | API调用超时/心跳断开 | 步骤按钮置灰，点击提示"咖啡机已离线，请检查网络连接" |
| 缺豆 | 咖啡机上报豆仓空 | 弹窗提示"咖啡豆不足，请添加后重试"，该步骤回退至"未开始" |
| 缺水 | 咖啡机上报水箱空 | 弹窗提示"请添加饮用水"，该步骤回退至"未开始" |
| 执行中断 | 设备状态异常/用户物理按键 | 同步状态为"已中断"，提示"制作已暂停，可重新启动" |
| 网络中断 | 请求失败 | Toast提示"网络异常，请检查后重试"，保留当前状态 |

## 4. 数据埋点

### 4.1 功能渗透指标
| 事件 | 说明 |
|:---|:---|
| `emotion_select_{mood}` | 选择情绪（mood= tired/happy/anxious/calm） |

### 4.2 推荐效果指标
| 事件 | 说明 |
|:---|:---|
| `emotion_recipe_show` | 推荐页曝光（含AI生成耗时） |
| `emotion_step_start_{n}` | 启动第n步 |
| `emotion_step_complete_{n}` | 第n步完成 |
| `emotion_step_stop_{n}` | 第n步主动停止 |
| `emotion_make_complete` | 完整制作流程结束 |

### 4.3 留存与商业指标
| 事件 | 说明 |
|:---|:---|
| `emotion_user_return_1d` | 使用后次日回访 |
| `emotion_user_return_7d` | 使用后7日回访 |
| `coffee_machine_use_cnt` | 咖啡机使用次数（对比实验组/对照组） |
| `emotion_session_duration` | 功能使用时长 |

## 5. 非功能需求

### 5.1 性能要求
- 页面加载：<2s（H5）
- AI推荐生成：<3s（含加载动画）
- 设备指令响应：<1s（本地网络）

### 5.2 兼容性
- 支持iOS 12+ / Android 8+ WebView
- 适配主流屏幕尺寸（320px-414px宽度）

### 5.3 安全
- 咖啡机控制指令需用户登录态校验
- 设备通信使用HTTPS/WSS加密

## 6. 附录

### 6.1 情绪-场景示例（供AI Prompt参考）

| 情绪 | 时段 | 天气 | 推荐方向 | 示例食谱 |
|:---|:---|:---|:---|:---|
| 疲惫 | 深夜 | - | 低因、温热、助眠 | 低因热拿铁·晚安版 |
| 疲惫 | 午后 | 雨天 | 温热、香甜、治愈 | 焦糖玛奇朵·雨天特调 |
| 开心 | 早晨 | 晴天 | 清爽、提神、仪式感 | 冰美式·晨间唤醒 |
| 焦虑 | 下午 | - | 低因、顺滑、安抚 | 燕麦拿铁·平缓心绪 |
| 想静一静 | 夜晚 | - | 手冲感、慢节奏、专注 | 单品手冲·独处时光 |

### 6.2 设备物模型要求
咖啡机需支持以下能力上报与指令接收：
- 研磨：研磨度、粉量
- 萃取：水温、压力、时间、水量
- 奶泡：温度、厚度（如支持）
- 保温：温度、时长
- 状态：空闲/工作中/异常+错误码
